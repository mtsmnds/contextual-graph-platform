# ADR Archive

## Purpose
Architecture Decision Records for significant project decisions.

## Naming
`YYYY-MM-DD-short-title.md`

## When Required
Changes affecting: build pipeline, schema/contracts, rendering, output conventions, agent/contributor process.

## Changelog Pairing
Significant `changelog.md` entries must include an ADR reference.

## Template
Copy `TEMPLATE.md` and fill: Context, Decision, Alternatives Considered, Consequences, Follow-ups.
